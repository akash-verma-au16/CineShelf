# CineShelf — App Data Files (v2.0.0)

All persistent data lives in a single directory managed by Electron's `userData` path:

```
C:\Users\<you>\AppData\Roaming\cineshelf\CineShelf\
├── settings.json       — user preferences, source dirs, API key, custom shelves
├── library.json        — file-system scan result (series + episode file paths)
├── metadata.json       — TMDB / TVMaze metadata cache (titles, overviews, cast, etc.)
├── history.json        — per-episode watch position and completion tracking
├── posters\            — downloaded poster images (.jpg), one per series/season
└── backdrops\          — downloaded backdrop images (.jpg), one per series/season
```

The path is determined in `electron/main.js`:

```js
const DATA_DIR = path.join(app.getPath('userData'), 'CineShelf');
```

## settings.json

**Written by:** `electron/main.js` → `saveSettings()`  
**Read by:** every startup via `loadSettings()`

Stores:

| Field             | Type       | Purpose                                                        |
| ----------------- | ---------- | -------------------------------------------------------------- |
| `sourceDirs`      | `string[]` | Folders to scan for TV series (e.g. `E:\`, `G:\`)              |
| `tmdbApiKey`      | `string`   | TMDB v3 API key for metadata fetching                          |
| `vlcPath`         | `string`   | Custom path to `vlc.exe` (blank = auto-detect)                 |
| `vlcHttpPort`     | `number`   | VLC HTTP interface port (default 8080)                         |
| `vlcHttpPassword` | `string`   | VLC HTTP interface password (default 'cineshelf')              |
| `autoResume`      | `boolean`  | Resume from saved position when reopening an episode           |
| `favorites`       | `string[]` | Series IDs marked as favourite                                 |
| `hallOfFame`      | `string[]` | Series IDs in the "Hall of Fame" tag                           |
| `highQuality`     | `string[]` | Series IDs tagged as high quality                              |
| `customRows`      | `object[]` | Home screen shelf rows — each has `{ id, title, seriesIds[] }` |

The `customRows` array is what drives the home screen layout. The fraction shown in the UI (`shelved / total`) compares unique IDs across all `seriesIds` arrays against the total scanned library.

## library.json

**Written by:** `electron/main.js` → startup scan + manual `library:scan` IPC  
**Read by:** `library:get` IPC on app load

The result of crawling all `sourceDirs`. Structure:

```json
{
	"lastScanned": "2026-03-13T10:00:00.000Z",
	"sourceDirs": ["E:\\", "G:\\"],
	"totalSeries": 32,
	"series": [
		{
			"id": "breaking-bad",
			"name": "Breaking Bad",
			"folderPath": "E:\\Breaking Bad",
			"sourceDir": "E:\\",
			"totalSeasons": 5,
			"totalEpisodes": 62,
			"addedAt": "2024-01-01T00:00:00.000Z",
			"seasons": [
				{
					"number": 1,
					"episodes": [
						{
							"id": "breaking-bad-s01e01",
							"seriesId": "breaking-bad",
							"season": 1,
							"episode": 1,
							"filename": "Breaking.Bad.S01E01.mkv",
							"filePath": "E:\\Breaking Bad\\Season 1\\Breaking.Bad.S01E01.mkv",
							"fileSize": 1234567890,
							"addedAt": "2024-01-01T00:00:00.000Z"
						}
					]
				}
			]
		}
	]
}
```

**Series IDs** are generated by `slugify(folderName)` in `electron/scanner.js` — lowercase, spaces → hyphens, special chars stripped. This ID is stable as long as the folder name doesn't change.

**Episode IDs** follow the pattern: `{seriesId}-s{season}e{episode}` e.g. `breaking-bad-s01e01`.

**Unavailable series:** when a source drive is disconnected, the last-known series entries are preserved in `library.json` with `unavailable: true`. They still render in the UI (greyed out) but cannot be played.

### How scanning works

1. App starts → waits 2 seconds → calls `runStartupScan()` in the background
2. Accessible source dirs are scanned; inaccessible ones are skipped
3. Series from inaccessible dirs are preserved from the previous scan with `unavailable: true`
4. `library.json` is overwritten with the fresh result
5. The renderer receives a `library:updated` IPC push

A manual rescan is triggered from the home screen or Settings page via `library:scan`.

## metadata.json

See [metadata-and-images.md](./metadata-and-images.md) for full detail.

A flat JSON object keyed by series ID:

```json
{
  "breaking-bad": {
    "id": "breaking-bad",
    "title": "Breaking Bad",
    "overview": "...",
    "genres": ["Drama", "Crime"],
    "year": "2008",
    "endYear": "2013",
    "rating": 9.5,
    "status": "Ended",
    "networks": ["AMC"],
    "cast": ["Bryan Cranston", "Aaron Paul", ...],
    "tmdbId": 1396,
    "numberOfSeasons": 5,
    "posterPath": "C:\\...\\posters\\breaking-bad.jpg",
    "backdropPath": "C:\\...\\backdrops\\breaking-bad.jpg",
    "seasons": {
      "1": {
        "name": "Season 1",
        "overview": "...",
        "airDate": "2008-01-20",
        "episodeCount": 7,
        "posterPath": "C:\\...\\posters\\breaking-bad_s1.jpg",
        "backdropPath": "C:\\...\\backdrops\\breaking-bad_s1.jpg"
      }
    },
    "episodes": {
      "1-1": {
        "title": "Pilot",
        "overview": "...",
        "runtime": 58,
        "airDate": "2008-01-20",
        "stillUrl": "https://image.tmdb.org/t/p/w300/..."
      }
    }
  }
}
```

Episode keys are `"{season}-{episode}"` (e.g. `"1-1"`, `"2-13"`).  
`runtime` is in minutes. Episode still images are **not** downloaded locally — they are served directly from TMDB URLs (`stillUrl`).

See [watch-history.md](./watch-history.md) for full detail.

```json
		"key": "breaking-bad-s01e01",
		"seriesId": "breaking-bad",
		"season": 1,
		"episode": 1,
		"position": 3480,

		"completed": true,
		"lastWatched": "2026-03-13T10:00:00.000Z"
	},

		"key": "series:breaking-bad",
		"seriesId": "breaking-bad",
		"completionCount": 1,
		"lastCompletedAt": "2026-03-13T10:00:00.000Z"
	}
}
```

## posters\ and backdrops\

Image files downloaded from TMDB (or TVMaze fallback) and stored locally.

**Naming convention:**

| File                   | What it is                      |
| ---------------------- | ------------------------------- |
| `{series-id}.jpg`      | Series-level poster or backdrop |
| `{series-id}_s{N}.jpg` | Season N poster or backdrop     |

All paths stored in `metadata.json` are absolute local paths to these files. The app never loads images from the internet at render time — everything is served via the `cineshelf:///` Electron protocol from local disk.

See [metadata-and-images.md](./metadata-and-images.md) for how images are fetched, stored, and updated.
