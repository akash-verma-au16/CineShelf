# CineShelf — Metadata & Images (v2.0.0)

## Overview

Metadata (titles, overviews, cast, episode info, ratings) and images (posters, backdrops) are fetched from external sources **once**, stored locally, and served from disk on every subsequent load. The app works fully offline after the initial fetch.

## Data Sources

### Primary: TMDB (The Movie Database)

- Requires an API key set in Settings (`tmdbApiKey`)
- Default key is bundled in the app but can be replaced with your own
- Rate limit: 40 requests per 10 seconds — the fetch code throttles at 260 ms per season to stay safe
- API base: `https://api.themoviedb.org/3`
- Image CDN: `https://image.tmdb.org/t/p`

### Fallback: TVMaze

- Free, no API key required
- Used automatically when TMDB has no result or when no API key is configured
- Does **not** provide backdrops or episode-level data — poster + basic show info only
- API: `https://api.tvmaze.com/search/shows`

## What Gets Fetched Per Series

When metadata is fetched for a series, the following TMDB endpoints are called in sequence:

| Step | Endpoint                     | What it gets                                                             |
| ---- | ---------------------------- | ------------------------------------------------------------------------ |
| 1    | `/search/tv`                 | Finds the show by name, returns `tmdbId`                                 |
| 2    | `/tv/{id}`                   | Full show details: overview, genres, cast, status, networks, season list |
| 3    | `/tv/{id}/credits`           | Cast list (top 12 names stored)                                          |
| 4    | `/tv/{id}/images`            | All available posters + backdrops with vote averages                     |
| 5    | `/tv/{id}/season/{n}`        | Per-season: episode list, air dates, runtimes, still URLs                |
| 6    | `/tv/{id}/season/{n}/images` | Season-level posters and backdrops                                       |

Image selection: best-voted poster and backdrop are chosen from the `/images` response (sorted by `vote_average` descending). Falls back to the default `poster_path`/`backdrop_path` from step 2 if no votes exist.

## Image Download & Storage

### Where images live

```
C:\Users\<you>\AppData\Roaming\cineshelf\CineShelf\
├── posters\
│   ├── breaking-bad.jpg          ← series poster
│   ├── breaking-bad_s1.jpg       ← season 1 poster
│   ├── breaking-bad_s2.jpg       ← season 2 poster
│   └── ...
├── backdrops\
│   ├── breaking-bad.jpg          ← series backdrop
│   ├── breaking-bad_s1.jpg       ← season 1 backdrop
│   └── ...
└── stills\
    ├── breaking-bad_s1e1.jpg     ← episode 1×01 still
    ├── breaking-bad_s1e2.jpg     ← episode 1×02 still
    └── ...
```

### Naming

Files are named `{safe-id}.jpg` or `{safe-id}_s{N}.jpg` where `safe-id` is generated by `safeId()` in `electron/metadata.js`:

```js
name.toLowerCase().replace(/[^\w]/g, '-').replace(/-+/g, '-');
```

This is similar to but slightly different from the library's `slugify()` — both produce stable kebab-case IDs from the series name.

### Download sizes

| Type                                 | TMDB size parameter | Approx resolution |
| ------------------------------------ | ------------------- | ----------------- |
| Series poster                        | `w500`              | 500 × 750 px      |
| Season poster                        | `w342`              | 342 × 513 px      |
| Series backdrop                      | `w1280`             | 1280 × 720 px     |
| Season backdrop                      | `w1280`             | 1280 × 720 px     |
| Season backdrop fallback (ep1 still) | `w780`              | 780 × 439 px      |
| Episode still (cached locally)       | `w780`              | 780 × 439 px      |

### "Exists" check — no redundant downloads

Before downloading any image, the code checks `fs.existsSync(dest)`. If the file already exists it is **not** re-downloaded. This means:

- Metadata re-fetch does **not** replace existing images
- To force a re-download, delete the file in `posters\` or `backdrops\` and re-fetch

### Episode still images

Episode stills are stored in two ways:

- **`stillUrl`** — the TMDB CDN URL at `w300` resolution, written into `metadata.json` during the initial metadata fetch. Used immediately in the UI as the CDN URL before local caching completes.
- **`stillLocalPath`** — absolute path to the locally-cached `w780` copy under `stills\`, written back into `metadata.json` by the background caching task once downloaded.

`EpisodeCard.js` prefers `stillLocalPath` (served via `cineshelf://`) when available, and falls back to `stillUrl` while the file hasn't been downloaded yet.

#### Background stills caching task

After the startup library scan and metadata fetch complete, `runStillsCachingTask()` runs automatically:

1. Checks `net.isOnline()` — exits silently if offline
2. Counts how many episodes have a `stillUrl` but are missing a local `w780` copy
3. If zero missing — exits immediately (no-op on subsequent startups)
4. Downloads each missing still from `https://image.tmdb.org/t/p/w780{still_path}` into `stills\{safeId}_s{season}e{episode}.jpg`
5. After each series completes, writes updated `metadata.json` (incremental — safe against restarts) and fires `metadata:patched` so the UI updates live
6. Fires `metadata:stills-cached` push event with `{ total, downloaded, skipped, failed }` when the full task is done

The task runs sequentially (not in parallel) and holds a `stillsCachingRunning` guard to prevent concurrent invocations. It can also be triggered manually via the `metadata:cache-stills` IPC channel (exposed in the renderer as `window.api.cacheStills()`).

Downloading at `w780` (rather than `w300`) ensures the stills can later be offered as per-episode backdrop candidates in the image picker.

## How Images Are Served in the App

All local images are served via a custom Electron protocol `cineshelf:///`:

```
cineshelf:///C:/Users/akash/AppData/Roaming/cineshelf/CineShelf/posters/breaking-bad.jpg
```

This avoids Chromium's security restrictions on `file://` URLs and supports range requests for video. The protocol handler is registered in `electron/main.js` using `protocol.handle('cineshelf', ...)`.

The renderer converts local paths to `cineshelf://` URLs using `toLocalUrl()` in `src/utils/helpers.js`.

## When Metadata Is Fetched

### Automatic (on startup)

After a library scan completes, `runStartupScan()` identifies series that have no metadata entry or are missing season data, and fetches them silently in the background. Results are pushed to the renderer via the `metadata:patched` IPC event.

### Manual (from UI)

- **Per-series:** clicking "Refresh Metadata" on the detail page calls `metadata:fetch-series`
- **Bulk:** Settings → "Fetch All Metadata" iterates all library series without complete metadata

---

## Updating Metadata

Fetching metadata again **overwrites** the metadata entry for that series in `metadata.json`. However:

- Images are **not** re-downloaded if the files already exist (see "Exists check" above)
- To get a newer/better image, use the image picker in the detail view (fetches all available alternatives from TMDB and lets you choose one to download and apply)

### Image picker (replacing images)

The detail view has an image picker that calls `metadata:fetch-image-alternatives` to list all available TMDB posters/backdrops for a series or season. Selecting one calls `metadata:set-image` which:

1. Downloads the chosen image to the correct path (overwriting the existing file)
2. Updates the `posterPath` / `backdropPath` in `metadata.json`
3. Pushes the updated metadata entry to the renderer via `PATCH_METADATA`

## Offline Behaviour

| Feature                  | Works offline? | Notes                                                                   |
| ------------------------ | -------------- | ----------------------------------------------------------------------- |
| Browse library           | ✅ Yes         | All series/episode data from `library.json`                             |
| View posters & backdrops | ✅ Yes         | Served from local disk via `cineshelf://`                               |
| View episode stills      | ✅ Yes\*       | Served locally once background caching has run; CDN fallback until then |
| Play episodes            | ✅ Yes         | Files served from local disk                                            |
| Fetch/refresh metadata   | ❌ No          | Requires TMDB API                                                       |
| Fetch image alternatives | ❌ No          | Requires TMDB API                                                       |
| Background stills cache  | ❌ No          | Checks `net.isOnline()` and skips if offline                            |

\* For series that have been in the library long enough for the background task to have run at least once while online.

## TVMaze Fallback Detail

If TMDB returns no results (e.g. no API key, or show not found), TVMaze is tried:

1. Search `https://api.tvmaze.com/search/shows?q={name}`
2. Download `show.image.original` (or `medium`) to `posters\`
3. Strip HTML from `show.summary` for the overview
4. Return basic fields: title, overview, year, genres, network, rating

TVMaze provides no backdrop, no season data, no episode runtimes. These fields remain empty until a TMDB fetch succeeds.
